**Load average ‚â† CPU usage**

Load average = number of processes that are:

* **Running on CPU**, or
* **Waiting for CPU**, or

## 3Ô∏è‚É£ Identify WHAT is contributing to the load

### üîç A. CPU bottleneck

```bash
top
```

Press:

* `1` ‚Üí per-CPU view
* `P` ‚Üí sort by CPU

Look for:

* `%us` high ‚Üí user processes
* `%sy` high ‚Üí kernel/syscalls
* `%wa` low ‚Üí not I/O related

Also:

```bash
mpstat -P ALL 1 5
```

**Root causes:**

* Infinite loops
* Too many threads
* Bad application logic
* Crypto / compression overload

---

### üîç B. I/O wait (very common cause)

```bash
top
```

If `%wa` is high ‚Üí disk or network storage problem.

Confirm with:

```bash
iostat -xz 1 5
```

Watch:

* `%util` near 100%
* `await` high (>20ms SSD, >50ms HDD)
* `svctm` vs `await` gap

Find offenders:

```bash
iotop -oPa
```

**Root causes:**

* Slow disks
* NFS latency
* Backup jobs
* Log flooding
* Database fsync storms

---

### üîç C. Memory pressure / swapping

```bash
free -h
vmstat 1 5
```

Red flags:

* Swap used heavily
* `si` / `so` non-zero
* Low `available` memory

Check who eats RAM:

```bash
ps aux --sort=-%mem | head
```

**Root causes:**

* Memory leak
* JVM heap mis-sizing
* Too many services
* Kernel cache pressure

---

### üîç D. Too many runnable processes

```bash
vmstat 1 5
```

Check:

* `r` column (run queue)

  * If `r` > CPU cores ‚Üí CPU contention

List process states:

```bash
ps -eo pid,stat,cmd | grep D
```

`D` state = uninterruptible sleep (usually I/O wait)

**Root causes:**

* Hung storage
* NFS freeze
* Blocked kernel calls

### Kernel logs

```bash

```

Look for:

* I/O errors
* Filesystem warnings
* OOM killer events
* NFS timeouts

## 5Ô∏è‚É£ Correlate with time (very important)

## 6Ô∏è‚É£ Common real-world root causes (from experience)

| Symptom                | Likely Root Cause   |
| ---------------------- | ------------------- |
| High load + low CPU    | Disk / NFS latency  |
| High load + swap       | Memory leak         |
| High load after reboot | Service misconfig   |
| Load spikes at night   | Backup / cron jobs  |
| Load stuck forever     | Hung filesystem     |
| High `%sy`             | Kernel bug / driver |

---

## 7Ô∏è‚É£ Root cause checklist (use this mentally)

Ask yourself:

* Did **CPU**, **disk**, or **memory** saturate first?
* Is load **runnable** or **blocked**?
* Is it **one process** or **many**?
* Did anything **change** (deploy, patch, config)?
